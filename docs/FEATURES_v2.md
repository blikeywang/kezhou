# 功能补充设计 v2 — 数据刷新 / 标的检索 / 收藏夹

配合 `BUILD_SPEC.md`。本文定义三项新功能的产品行为与工程实现要点。原型 `prototype/app.html` 已做可点演示(前端模拟)。

---

## A. 数据刷新(时间前滚 → 数据要跟上)

**目标**:时间每天推进,分析必须基于最新已收盘的 K 线;同时严格控制抓取速率、控制成本。

**三层策略**:

1. **服务端增量(权威源)** — 定时任务按周期把新 K 线追加入库(4h 每 4 小时、日线每交易日收盘后、周线每周)。这是数据的**唯一权威来源**,面向全体用户,不受单用户触发。用正规数据商 + 退避重试;失败告警。
2. **按需 fetch(冷门/新标的)** — 库里没有或已过期的标的,在**用户查询时**触发拉取,但必须限速:
   - **服务端限流**:每个数据源全局令牌桶(如 Binance/数据商各自 QPS 上限),排队 + 退避;命中限流返回"排队中,请稍候"。
   - **去重**:同标的并发请求合并(single-flight),只发一次外部请求,多个用户共享结果。
   - **缓存/TTL**:结果按 `(symbol,timeframe,交易日)` 缓存;当日复用,收盘后失效。原型踩过的 Binance IP 封禁就是缺这层限速导致的(见 BUILD_SPEC §3.1),生产务必做。
   - **冷启动**:新标的首次查询先补一段历史(如日线 15y、4h 3y)再算,之后转增量。
3. **每月推送常用数据** — 面向登录用户,把其**常看/收藏**标的的最新数据在月初批量预热(预抓 + 预算匹配 + 预热缓存),并可发一封"本月数据已更新"的邮件/站内信。好处:用户月初打开常用标的时**秒开**、且我们把抓取放在可控的批处理窗口而非高峰随机触发。
   - 实现:每月定时任务遍历"活跃用户 × 收藏/高频标的"去重集合 → 分批限速预热 → 记录 `refreshed_at`。
   - 频率可配:核心标的日更,长尾标的月更或按需。

**新鲜度可见**:每个分析页显示"数据截至 {日期}",并给手动"刷新"(受同一限速保护:同标的 N 分钟内只允许一次,过频提示限速)。UI 用徽标区分 实时/日更/月更/按需。

## B. 标的检索 + 存在性校验 + 用户确认

**目标**:让爱搞小众标的的用户能搜到并加载,同时避免"代码打错/歧义/不存在"造成的无效抓取与困惑。

**流程(两步确认)**:

1. **检索与校验** — 用户在搜索框输入(代码或名称)。前端做**自动补全**(打 `/symbols/search?q=` 命中我们维护的标的字典:代码、名称、交易所、所属战场、别名)。选中或回车后,服务端**确认标的是否存在**:
   - 命中字典 → 直接确认;
   - 未在字典但疑似有效 → 向数据商做一次**存在性探测**(轻量 meta 请求,不拉全量);
   - **与用户确认**:弹出确认卡"你是指 **台积电 TSM(NYSE · 美股)** 吗?" 让用户点确认,消除歧义(如 TSM 可能是台积电也可能是别的)。
2. **确认后按查询加载** — 用户确认且标的存在:
   - 若库中已有 → 直接查询;
   - 若库中没有 → 归入 **B 层按需 fetch**(限速 + 冷启动补历史)→ 拉到数据后运行匹配 → 展示。此次查询照常**消耗当日额度**(把"新标的加载"计为一次查询)。
   - 加载成功后把该标的纳入常规增量更新与(若用户收藏)月度预热。

**边界**:不存在/退市/数据不足 → 明确提示,不消耗额度或消耗后可退回;地域/合规不可提供的标的按黑名单拦截。

**接口草案**:
- `GET /symbols/search?q=&market=` → 候选列表(补全用)。
- `POST /symbols/resolve {query}` → `{exists, canonical:{ticker,name,exchange,market}, ambiguous?[候选]}`(存在性 + 确认信息)。
- `GET /analysis/{symbol}?timeframe=&window=` → 若需按需 fetch 则内部排队,返回结果或"排队中"。

## C. 收藏夹 / 自选(多命名列表 + 本地缓存)

**目标**:用户把爱看的标的收藏起来,可分多个可命名的列表;进入收藏的内容自动缓存到本地,打开更快;跨设备可同步。

**数据模型**:
```
watchlist { id, userId, name, order, createdAt }
watchlist_item { watchlistId, symbol, market, addedAt }
```
默认给每个用户一个"我的自选";用户可**新建、命名、重命名、删除、拖拽排序**多个列表;一个标的可在多个列表。

**本地缓存(离线秒开)**:
- 收藏项的**最近一次分析结果**(数值 payload,非大图)写入浏览器 **IndexedDB**(比 localStorage 容量大、结构化;localStorage 仅存轻量偏好)。
- 打开收藏项时**本地优先**渲染(带"数据截至 X"),同时后台按限速拉取增量,新数据到达后静默更新并刷新徽标(stale-while-revalidate)。
- 缓存带 TTL 与版本号;引擎算法或数据修订时版本号变更 → 本地失效重取。
- 隐私:本地缓存只存该用户自己的数据;登出可清除。

**同步**:列表与成员存服务端(跨设备一致);结果缓存在本地(省流量、快)。离线可看已缓存的收藏,联网后补新。

**与月度推送联动**:收藏标的自动进入 B 层月度预热名单 → 月初这些标的服务端已算好,用户本地缓存也在打开时快速对齐。

**接口草案**:
- `GET/POST/PATCH/DELETE /watchlists`、`/watchlists/{id}/items` — 增删改查与排序。
- 前端:IndexedDB 封装 `cache.get(symbol)/put(symbol,payload,ver)`;收藏页本地优先 + 后台 revalidate。

## D. 逐年同期对比(覆盖经济/行业周期)

**目标**:季节性不能只看"去年同期"——很多行业有经济/行业周期,单一年份会误导。

- 分析页在"同期季节性(汇总)"下方增加**逐年同期对比表**:列出**过去每一年**同一日历日之后在近/中/远前瞻期的收益(现有数据加密约 8 年、美股/大宗约 15 年,均 ≥5 年,跨多轮周期)。
- 汇总行给上涨年占比与中位;逐年行让用户自行判断该形态在不同年份/环境下是否稳定。
- 生产:季节性统计随数据日更;样本年数在 UI 标明,并提示"样本有限,仅看方向"。

## E. 讲解层(新手引导 + 老用户可隐藏)

**目标**:不是甩给用户一张表说"这是什么"就完事——每块数据都配清楚的解释;新手默认看到,老用户可一键关闭/隐藏。

- 顶栏「讲解」开关(默认**开**,面向新用户);每个区块(KPI、概率表、逐年季节性、相似片段、形态)下方有**讲解框**,用大白话说明每个数字/每一列怎么读、怎么对比基准、显著性是什么。
- 任意讲解框可点"隐藏说明 ✕"一次性关闭;状态随账号持久化(`explain`),老用户不再被打扰。
- 生产:讲解文案走 i18n;可做成分级(新手全开 / 进阶只留提示图标 / 专家全关);首次进入可加一条引导 tooltip 串讲。

## F. 内容分层(免费只看一部分,防白嫖)

**目标**:免费用户看到足够"值"的引流内容,但**深度数据锁定给 Pro**,避免每天被白嫖。与每日额度(免费 5 / Pro 50)构成**双闸**。

**免费可见(引流)**:形态匹配全景图、方向结论(偏多/偏空)、核心 KPI(中期上涨概率 + 基准)、形态拆解一句话。够看懂"这个标的现在像历史上哪种、大致偏多还是偏空"。

**Pro 解锁(锁定给免费用户)**:
- 完整**前瞻概率表**(各周期 corr/DTW/基准/中位/分位)——最核心的量化数字。
- **逐年同期对比表**(跨周期的逐年明细)。
- **最相似历史片段**(可回溯的具体日期 + 回撤)。
- 多周期/多窗口对照、片段导出、按需加载更多长尾标的。

**实现要点**:
- **服务端裁剪**:免费用户的 API 只返回可见字段(图 + 结论 + 概览),锁定字段根本不下发——不能只靠前端隐藏(否则可从接口/DOM 抓到)。前端对锁定块渲染"升级解锁"卡。
- 付费墙 CTA 记录 `upgrade_click{context:"lock", feature}`,便于分析哪块内容最能驱动付费(见 ANALYTICS.md)。
- 可做"每日 1 次完整预览"之类的钩子提高转化(可选)。

> 原型:免费用户下,概率表/逐年/相似片段显示为**升级解锁卡**,点击弹升级;切到 Pro(升级演示)后立即解锁。

## 交互增量(已在原型演示)
- 侧栏顶部**搜索框**:输入即过滤当前战场品类;输入库外代码 → "搜索『XXX』" → 存在性**确认卡** → 确认后加载(计一次额度)。
- **收藏夹区**:多命名列表切换/新建/重命名;每个品类与分析页有 ★ 星标一键收藏;收藏项标注"已本地缓存"。
- **新鲜度徽标**:分析页"数据截至 {日期} · 刷新";刷新受限速保护(过频提示)。
- 概览提示:"常用标的每月自动更新"。
- **逐年同期对比表**:分析页展示过去每一年同期之后的收益(≥5 年,跨周期)。
- **讲解层**:顶栏「讲解」开关(默认开),每块数据配大白话说明,老用户可一键隐藏并记忆。

> 原型中的检索字典、存在性校验、按需 fetch、限速与本地缓存均为**前端模拟**,用于演示交互;生产实现按上文 A/B/C 落地。
